# 1. 스프라이트 시트 (Sprite Sheet)
스프라이트 시트는 여러 개의 작은 이미지(프레임)를 하나의 큰 파일에 모아둔 것입니다.

동작 원리: 전체 시트 이미지에서 필요한 부분만 좌표(x, y)와 크기(width, height)를 지정해 잘라서 사용합니다.

중요성: 수많은 이미지 파일을 따로 불러오는 것보다 시스템 자원을 훨씬 적게 사용하며, 애니메이션(예: 걷는 동작)을 구현할 때 프레임 관리가 매우 효율적입니다.

활용: subsurface() 함수로 자른 뒤, transform.scale()로 게임 내 크기에 맞게 조절하여 사용합니다.

# 2. 오디오 적용 (Audio Integration)
게임의 몰입감을 결정하는 요소로, 배경음(BGM)과 효과음(SFX)으로 나뉩니다.

BGM (배경음): 보통 pygame.mixer.music을 사용하며, 파일 크기가 크기 때문에 스트리밍 방식으로 재생하고 무한 반복(-1) 설정을 주로 합니다.

SFX (효과음): 먹이를 먹거나 충돌하는 순간 등 짧은 소리에 사용됩니다. pygame.mixer.Sound 객체로 미리 메모리에 로드해두었다가 필요할 때 지연 없이 play() 합니다.

핵심: 사운드 채널 설정을 통해 여러 소리가 겹치더라도 끊기지 않게 관리하는 것이 기술적 포인트입니다.

# 3. 카메라 추적 (Camera Tracking)
플레이어(뱀)가 화면 중앙에 머물도록 월드 전체를 움직이는 기법입니다.

동작 원리: 카메라가 실제로 움직이는 것이 아니라, **플레이어의 위치와 화면 중앙의 차이(Offset)**만큼 다른 모든 오브젝트(벽, 먹이, 배경)를 반대 방향으로 밀어서 그립니다.

중요성: 맵이 화면 크기보다 훨씬 넓을 때 플레이어가 길을 잃지 않고 탐험할 수 있게 해줍니다.

코드 적용: 모든 blit이나 draw 함수에서 좌표에 cam_x, cam_y 변수를 더해주는 방식으로 구현합니다.

# 4. 무적 프레임 (Invincibility Frames / Ghost Mode)
특정 아이템을 먹거나 피해를 입었을 때 일시적으로 충돌 판정을 무시하는 상태입니다.

동작 원리: 현재 시간(pygame.time.get_ticks())과 무적 종료 시간을 비교하는 타이머 로직을 사용합니다.

시각적 피드백: 플레이어가 무적 상태임을 알 수 있도록 캐릭터를 반투명하게 만들거나(Alpha 값 조절), 색상을 변경하는 연출이 필수적입니다.

중요성: 게임의 난이도를 조절하고, 플레이어에게 위기를 탈출하거나 전략적인 플레이를 할 수 있는 기회를 제공합니다.

# 5. 패럴랙스 스크롤링 (Parallax Scrolling)
배경의 레이어마다 이동 속도를 다르게 설정하여 2D 게임에서 3D 같은 공간감을 주는 기법입니다.

동작 원리: 카메라가 움직일 때 가까운 배경은 빠르게, 먼 배경(별이나 산)은 아주 천천히 움직이게 배율(0.2x, 0.4x 등)을 곱합니다.

효과: 단순한 평면이었던 배경에 깊이감(Depth)이 생겨 게임이 훨씬 고급스러워 보입니다.

활용: 현재 코드에서는 stars_far와 stars_mid에 서로 다른 배율을 적용하여 우주 공간의 깊이감을 표현하고 있습니다.
