## week06


# 1. 경로 지정 방식: 절대 경로 vs 상대 경로
절대 경로 (Absolute Path)
정의: "C:\Users\Documents\image.png 처럼 파일의 전체 주소를 처음부터 끝까지 다 적는 방식입니다.

문제점: 내 컴퓨터에서는 잘 되지만, 다른 사람의 컴퓨터로 파일을 옮기면 경로가 달라져 에러가 납니다. 또한, 백슬래시(\) 가 많아 r 을 붙여야 하는 번거로움이 있습니다.

상대 경로 (Relative Path) — 추천 방식
정의: 현재 실행 중인 파이썬 파일(.py)의 위치를 기준으로 파일 주소를 찾는 방식입니다.

기호 설명:

./ : 현재 내가 있는 폴더 (생략해도 무방합니다.)

../ : 현재 폴더의 한 단계 상위 폴더

장점: 프로젝트 폴더 자체를 통째로 옮겨도 경로 수정 없이 그대로 실행됩니다. \ 대신 /를 쓰면 r 을 붙일 필요도 없어 코드가 매우 깔끔해집니다.

예시: pygame.image.load("image.png") 또는 pygame.image.load("./assets/bg.png")

# 2. 이미지 최적화와 convert_alpha()
이미지를 불러올 때는 눈에 보이는 것 이상의 데이터 처리가 필요합니다.

최적화 이유: PNG 이미지의 투명한 배경을 유지하고, 게임 화면을 그리는 속도를 극대화하기 위해서입니다.

효과: convert_alpha()를 쓰지 않으면 투명해야 할 부분이 검은 박스로 보이거나, 화면이 뚝뚝 끊기는 프레임 드랍(FPS 저하) 현상이 생길 수 있습니다.

# 3. Rect 객체를 활용한 정교한 배치
이미지를 화면 어디에 놓을지 정할 때, Pygame의 Rect(사각형) 객체는 필수입니다.

좌표계: 왼쪽 위가 (0, 0)이며, 오른쪽은 x 증가, 아래쪽은 y 증가입니다.

기준점 배치:

rect.topleft: 왼쪽 상단 구석 기준 배치

rect.center: 이미지의 정중앙 기준 배치

rect.midbottom: 하단 중앙 기준 배치 (땅 위에 서 있는 캐릭터에 적합)

# 4. 애니메이션 및 화면 경계 처리 (Screen Wrapping)
물체를 움직이고 화면 끝을 제어하는 로직입니다.

이동: 루프 내에서 좌표를 계속 변경합니다. (rect.x += 2, rect.y += 2 등)

스크린 래핑: 물체가 화면 밖으로 나갔을 때 반대편에서 나타나게 하는 기술입니다.

로직: 만약 물체의 왼쪽 끝(rect.left)이 화면 너비를 넘어가면, 물체의 오른쪽 끝(rect.right)을 다시 시작점인 0 으로 옮겨줍니다. 이렇게 하면 자연스럽게 순환하는 움직임이 완성됩니다.

# 5. 사운드 및 시스템 제어
배경음악과 효과음은 목적에 따라 재생 방식이 나뉩니다.

배경음악 (BGM): pygame.mixer.music을 사용하며, 페이드인(Fade-in) 효과(fade_ms)를 주면 0 부터 설정한 볼륨까지 서서히 소리가 커지며 시작됩니다.

효과음 (SFX): pygame.mixer.Sound를 사용합니다.

리셋 기능: 같은 소리가 겹쳐서 시끄러워지는 것을 방지하려면, 재생 전 stop()을 먼저 호출한 뒤 play()를 실행하여 항상 처음부터 소리가 나오도록 제어합니다.

# 6. 사용자 이벤트 (키보드 및 마우스)
키보드: pygame.KEYDOWN으로 특정 키 입력을 감지합니다. (ESC로 종료 등)

마우스: pygame.MOUSEBUTTONDOWN으로 클릭을 감지합니다.

event.button == 1 을 통해 왼쪽 클릭만 골라낼 수 있으며, event.pos로 클릭된 정확한 위치(x, y)를 받아와 게임 로직에 활용할 수 있습니다.
